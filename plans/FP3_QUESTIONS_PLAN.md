# FP3級タイピング問題刷新プラン

作成日: 2026-04-30

## 結論

難易度は文字数だけで決めない。  
`文字数`、`FP概念の難度`、`入力しづらさ` の3軸で設計する。

画面には日本語のFP用語・短文を表示し、入力はローマ字にする。

```json
{
  "category": "ライフプランニングと資金計画",
  "display": "公的年金",
  "input": "kouteki nenkin"
}
```

現在の実装は文字列配列を前提にしているため、上記形式に移行する場合は `script.js` の問題読込・表示・判定処理も合わせて変更する。

## 参照範囲

FP3級の問題範囲は、公式の6分野を軸にする。

- A. ライフプランニングと資金計画
- B. リスク管理
- C. 金融資産運用
- D. タックスプランニング
- E. 不動産
- F. 相続・事業承継

参照:

- 日本FP協会: https://www.jafp.or.jp/exam/subjects_03/
- 金融財政事情研究会: https://www.kinzai.or.jp/fp/fp_specifications.html

## 難易度設計

### Level 1: 用語単体

目的: FP3級の基本語彙をタイピングしながら覚える。

条件:

- 日本語表示は1語
- 入力はローマ字1語中心
- 記号・数字は原則なし
- 文字数目安: 4〜18文字

例:

```json
{ "category": "リスク管理", "display": "生命保険", "input": "seimei hoken" }
{ "category": "不動産", "display": "建ぺい率", "input": "kenpeiritsu" }
{ "category": "相続・事業承継", "display": "遺留分", "input": "iryuubun" }
```

目標問題数:

- 6分野 x 60問 = 360問
- 最低ライン: 300問

### Level 2: 複合用語

目的: 試験でよく出る用語の組み合わせに慣れる。

条件:

- 日本語表示は2〜4語程度
- 入力はスペース区切りを含める
- 文字数目安: 10〜28文字
- 似た用語を混ぜる

例:

```json
{ "category": "ライフプランニングと資金計画", "display": "老齢基礎年金", "input": "rourei kiso nenkin" }
{ "category": "金融資産運用", "display": "投資信託の分配金", "input": "toushi shintaku no bunpaikin" }
{ "category": "タックスプランニング", "display": "所得控除と税額控除", "input": "shotoku koujo to zeigaku koujo" }
```

目標問題数:

- 6分野 x 50問 = 300問
- 最低ライン: 240問

### Level 3: 短い説明文

目的: FP用語を文脈つきで覚える。

条件:

- 日本語表示は短文
- 入力はローマ字文
- 文字数目安: 25〜55文字
- 数字・税率・年齢など、法改正で変わりやすい値は避ける

例:

```json
{
  "category": "リスク管理",
  "display": "生命保険は死亡や高度障害に備える保険である",
  "input": "seimei hoken wa shibou ya koudo shougai ni sonaeru hoken de aru"
}
```

目標問題数:

- 6分野 x 35問 = 210問
- 最低ライン: 180問

### Level 4: 試験文風の長文

目的: 試験文に近い語彙密度の文章を入力する。

条件:

- 日本語表示は1文
- 入力はローマ字文
- 文字数目安: 50〜90文字
- 断定しすぎる制度説明は避ける
- 変わりやすい金額・税率・年齢は使わない

例:

```json
{
  "category": "相続・事業承継",
  "display": "相続対策では遺産分割、納税資金、相続税評価を総合的に確認する",
  "input": "souzoku taisaku dewa isan bunkatsu nouzei shikin souzokuzei hyouka wo sougouteki ni kakunin suru"
}
```

目標問題数:

- 6分野 x 25問 = 150問
- 最低ライン: 120問

## 分野別配分

各レベル内で6分野を均等に入れる。

理由:

- FP3級の学習範囲を偏らせない
- 難易度選択だけでなく、将来的に分野別練習へ拡張しやすい
- 問題データのレビュー時に漏れを見つけやすい

推奨カテゴリキー:

```json
[
  "life_planning",
  "risk_management",
  "financial_assets",
  "tax_planning",
  "real_estate",
  "inheritance"
]
```

## データ形式

現在の `questions_level*.json` は文字列配列。

刷新後はオブジェクト配列にする。

```json
[
  {
    "id": "life_l1_001",
    "level": 1,
    "category": "life_planning",
    "display": "公的年金",
    "input": "kouteki nenkin",
    "tags": ["nenkin", "basic"]
  }
]
```

必須:

- `id`: 安定した問題ID
- `level`: 1〜4
- `category`: 6分野のカテゴリキー
- `display`: 画面に表示する日本語
- `input`: 入力対象のローマ字

任意:

- `tags`: 後から検索・分野別練習に使う

## ローマ字ルール

統一ルールを先に決める。

- 長音は基本的に母音重ね: `koujo`, `yuushi`, `hyouka`
- 助詞の「を」は `wo`
- 助詞の「は」は読み優先で `wa`
- 助詞の「へ」は読み優先で `e`
- 促音は子音重ね: `kessan`, `shutoku`
- スペースは語の区切りにだけ使う
- 記号は原則使わない
- Level 4のみ、必要ならカンマやピリオドを検討する

## 問題作成ルール

入れる:

- 公式範囲に出てくる基本語
- 市販テキストの章立てに近い一般用語
- 法改正に左右されにくい概念
- 試験で頻出しやすい対比語

避ける:

- 最新税率、控除額、年齢、期限などの数値暗記
- 特定年度の制度改正に依存する文
- 正誤判定が必要な文章
- 専門的すぎる2級以上の論点
- 入力が極端に長いカタカナ語

## 実装ステップ

1. 問題データのスキーマを確定する
2. `questions_level1.json` から `questions_level4.json` をオブジェクト配列へ移行する
3. `script.js` を `display` と `input` に対応させる
4. 画面に `display` を表示し、入力判定は `input` に対して行う
5. 結果画面に分野別の入力数を出せるよう、内部状態に `category` を保持する
6. 各JSONの件数・ID重複・必須キー・ローマ字文字種を検証するスクリプトを追加する
7. ブラウザで各レベルの読み込み、入力、次キー表示、結果表示を確認する

## 検証方針

最低限の自動検証:

- JSONとして正しい
- 配列である
- 必須キーがある
- `id` が重複していない
- `level` がファイル名と一致している
- `category` が6分野のいずれか
- `input` が半角英小文字、スペース、必要最小限の記号だけ
- `display` が空でない

手動検証:

- Level 1〜4それぞれ開始できる
- 日本語表示とローマ字入力が対応している
- スペースを含む問題を入力できる
- 長文で画面が崩れない
- 次キー表示が空白や記号で破綻しない
- 全問消費時に結果画面へ遷移する

## 実装時の注意

- 初回は問題生成とUI改修を分ける
- まず20問程度のサンプルでスキーマ移行を確認する
- 動作確認後に大量データを投入する
- 大量問題投入後はJSON検証を必須にする
- 問題文の正確性レビューは実装レビューとは別に行う
